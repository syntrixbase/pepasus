---
name: review
description: Use when reviewing code changes, PRs, or asking for code feedback
context: fork
argument-hint: "[pr-number or description]"
---

Review code changes for $ARGUMENTS:

1. Identify what changed (git diff, file list)
2. Check for bugs, logic errors, edge cases
3. Check for security issues
4. Evaluate test coverage
5. Summarize findings with specific file and line references

Focus on substance, not style. Flag only issues that matter.
